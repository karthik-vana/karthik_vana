## 🌟 Key Features

- **🌌 3D Galaxy Background**: Immersive, interactive particle system using **Three.js** & **React Three Fiber**.
- **🖱️ Custom Spider Cursor**: Unique, physics-based cursor interaction for engagement.
- **🍱 Bento Grid Layout**: Modern, responsive grid design for showcasing content eleganty.
- **🌊 Smooth Scrolling**: Integrated **Lenis** for buttery smooth scroll experiences.
- **✨ Premium Animations**: Powered by **Framer Motion** and **GSAP** for world-class transitions.
- **📱 Fully Responsive**: Flawless experience across all devices (Mobile, Tablet, Desktop).

---

## 🛠️ Tech Stack

Built with cutting-edge technologies to ensure performance and scalability:

- **Core**: [React 19](https://react.dev/), [Vite](https://vitejs.dev/)
- **Styling**: [Tailwind CSS](https://tailwindcss.com/), [Sass](https://sass-lang.com/)
- **3D & Graphics**: [Three.js](https://threejs.org/), [React Three Fiber](https://docs.pmnd.rs/react-three-fiber), [Drei](https://github.com/pmndrs/drei)
- **Animation**: [Framer Motion](https://www.framer.com/motion/), [GSAP](https://greensock.com/gsap)
- **UX**: [Lenis](https://github.com/studio-freight/lenis) (Smooth Scroll)

## 🚀 Getting Started

Follow these steps to set up the project locally.

### 1. **Clone the Repository**
```bash
git clone https://github.com/yourusername/portfolio-v.git
cd portfolio-v
```

### 2. **Install Dependencies**
Install all necessary packages using npm:
```bash
npm install
```

### 3. **Run Development Server**
Start the local development server:
```bash
npm run dev
```
> The app will run at `http://localhost:5173`.

---

## 📦 Build & Deploy

To create an optimized production build:

```bash
npm run build
```
This will generate the `dist` folder, ready for deployment on **Vercel**, **Netlify**, or **GitHub Pages**.

## 🔄 Git Workflow (How to Push)

To save your changes and push them to the repository, follow these best practices:

1. **Check Status**: See which files have changed.
   ```bash
   git status
   ```

2. **Stage Changes**: Add files to the staging area.
   ```bash
   git add .
   ```
   *(Use `git add .` to add all changes, or specify files individually)*

3. **Commit Changes**: Save your snapshot with a meaningful message.
   ```bash
   git commit -m "feat: added new animation section"
   ```

4. **Push to Remote**: Upload your code to GitHub.
   ```bash
   git push origin main
   ```
   *(Replace `main` with your branch name if different)*
